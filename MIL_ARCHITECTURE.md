# 多示例学习（MIL）架构设计文档

## 一、概述

多示例学习（Multi-Instance Learning, MIL）是一种弱监督学习范式，核心思想是：
- **包（Bag）**：包含多个实例的集合（如一张遥感影像）
- **实例（Instance）**：包中的单个元素（如影像中的一个像素块）
- **标签**：只给包级别的标签（图像级标签），不给实例级标签（像素级标签）

本文档描述如何将MIL集成到现有的FusionCropNetV5系统中。

## 二、整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MIL集成架构                                      │
├─────────────────────────────────────────────────────────────────────┤
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐          │
│  │   数据层     │───→│   模型层     │───→│   训练层     │          │
│  │ MIL数据集    │    │ MIL模块      │    │ MIL训练流程  │          │
│  └──────────────┘    └──────────────┘    └──────────────┘          │
│         │                    │                    │                 │
│         ↓                    ↓                    ↓                 │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │                      FusionCropNetV5MIL                      │   │
│  │  [光学/SAR/DEM编码器] → [时序Transformer] → [MIL注意力池化]   │   │
│  └──────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

## 三、核心技术组件

### 3.1 数据层设计

| 组件 | 功能 | 技术细节 |
|------|------|----------|
| **MIL数据集** | 管理包-实例结构 | 支持滑动窗口切分、动态实例数量 |
| **实例提取器** | 从影像中提取斑块 | 可配置patch_size和overlap |
| **标签映射器** | 图像级标签→类别索引 | 支持灵活的标签体系 |

**关键设计**：
- 支持**动态实例数量**（不同影像切分出的斑块数量不同）
- 支持**多尺度实例**（可同时提取不同大小的斑块）
- 支持**弱监督增强**（随机丢弃部分实例模拟噪声）

### 3.2 模型层设计

| 组件 | 功能 | 技术细节 |
|------|------|----------|
| **MIL注意力池化** | 学习实例重要性权重 | 多头自注意力机制 |
| **实例特征提取** | 提取每个实例的特征 | 复用现有编码器 |
| **包级别分类头** | 基于聚合特征分类 | 线性层+激活函数 |
| **注意力可视化** | 展示实例重要性 | 热力图可视化 |

**注意力机制设计**：
```
输入: [B, N, C, P, P]  → 实例特征提取 → [B, N, D]
                                       ↓
                           MIL注意力池化 (多头自注意力)
                                       ↓
输出: [B, D] (包特征) + [B, N] (注意力权重)
```

### 3.3 训练策略设计

| 策略 | 说明 | 技术细节 |
|------|------|----------|
| **两阶段训练** | 先强监督预训练，再MIL微调 | 迁移学习策略 |
| **混合监督** | 同时使用强监督和弱监督数据 | 多任务学习 |
| **自监督辅助** | 利用无标签数据预训练 | 对比学习、掩码建模 |
| **伪标签生成** | 用强监督模型生成伪标签 | 半监督学习 |

## 四、与现有系统的集成

### 4.1 模型集成

```
FusionCropNetV5MIL (继承自FusionCropNetV5EDL)
├── 共享组件：光学编码器、SAR编码器、DEM编码器、时序Transformer
├── 新增组件：MIL注意力池化、包级别分类头
└── 可选组件：EDL不确定性估计（保留原有的EDL功能）
```

### 4.2 数据集成

```
现有数据集
├── 强监督数据（像素级标签）→ 用于预训练
└── MIL数据（图像级标签）→ 用于MIL微调
```

### 4.3 训练集成

```
训练脚本
├── train_fusion.py → 强监督训练（现有）
├── train_mil.py → MIL训练（新增）
└── train_hybrid.py → 混合监督训练（新增）
```

## 五、关键技术细节

### 5.1 实例提取策略

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| patch_size | 32/64 | 实例大小 |
| overlap | 0.5 | 重叠率，平衡计算量和信息保留 |
| stride | patch_size × (1-overlap) | 滑动步长 |

### 5.2 注意力机制选择

| 方法 | 复杂度 | 效果 | 适用场景 |
|------|--------|------|----------|
| 平均池化 | O(1) | 简单，无学习 | 基线对比 |
| 最大池化 | O(N) | 关注最显著实例 | 正例稀疏场景 |
| 注意力池化 | O(N²D) | 学习实例重要性 | 推荐方案 |
| 可学习权重 | O(ND) | 线性复杂度 | 大规模数据 |

### 5.3 损失函数设计

| 损失类型 | 说明 |
|----------|------|
| 包级别CE | 标准MIL损失，基于包级别的交叉熵 |
| 实例级别CE | 带注意力权重的实例级别损失 |
| 对比损失 | 自监督预训练使用 |

### 5.4 不确定性估计集成

保留EDL不确定性估计功能：
```
MIL推理流程
├── 包级别预测
├── 实例级别注意力权重
└── 不确定性估计（基于EDL）
    ├── Vacuity（包级别）
    └── Dissonance（包级别）
```

## 六、性能优化策略

| 优化方向 | 方法 | 效果 |
|----------|------|------|
| **计算效率** | 实例特征批量提取 | 减少前向传播次数 |
| **内存优化** | 梯度检查点 | 减少显存占用 |
| **训练加速** | 混合精度训练 | 提升训练速度 |
| **数据效率** | 实例采样策略 | 只处理关键实例 |

## 七、评估指标

| 指标类型 | 指标 | 说明 |
|----------|------|------|
| **包级别** | Accuracy, F1, AUC | 评估图像级分类效果 |
| **实例级别** | mIoU, OA | 评估像素级分类效果 |
| **注意力质量** | 注意力熵 | 衡量注意力分布 |
| **不确定性** | Brier分数 | 评估不确定性校准 |

## 九、总结

多示例学习的核心价值在于**利用图像级标签进行弱监督学习**，可以大幅减少标注成本。其关键技术包括：

1. **包-实例结构**：将影像视为包，斑块视为实例
2. **注意力机制**：学习哪些实例对分类更重要
3. **两阶段训练**：先强监督预训练，再MIL微调
4. **与EDL集成**：保留不确定性估计能力

这种架构可以很好地集成到现有的FusionCropNetV5系统中，无需大规模重构。
