## 二、模块功能详解

### 1. API网关层 (FastAPI)

| 功能 | 说明 |
|------|------|
| 路由管理 | 定义API端点，映射请求到处理函数 |
| 请求验证 | 使用Pydantic模型验证输入数据 |
| 响应序列化 | 自动将Python对象转换为JSON |
| CORS配置 | 跨域资源共享配置 |
| 异常处理 | 统一的错误处理和HTTP状态码 |
| 文档自动生成 | 自动生成Swagger UI和ReDoc文档 |

### 2. 推理服务模块

| API端点 | 方法 | 功能 |
|---------|------|------|
| `/inference` | POST | 单样本推理，支持不确定性估计 |
| `/inference/batch` | POST | 批量推理，处理多个样本 |
| `/inference/file` | POST | 文件上传推理 |

### 3. 模型管理模块

| API端点 | 方法 | 功能 |
|---------|------|------|
| `/model/status` | GET | 获取模型加载状态 |
| `/model/load` | POST | 加载指定模型权重 |
| `/model/unload` | POST | 卸载当前模型 |
| `/model/list` | GET | 列出可用模型 |

### 4. 系统管理模块

| API端点 | 方法 | 功能 |
|---------|------|------|
| `/health` | GET | 健康检查 |
| `/version` | GET | 获取版本信息 |
| `/metrics` | POST | 计算模型指标 |

### 5. 业务逻辑层

| 组件 | 功能 |
|------|------|
| ModelLoader | 负责模型的加载和管理 |
| PreprocessPipe | 数据预处理管道 |
| MetricCalculator | 指标计算（mIoU、OA等） |

### 6. 监控与日志层

| 组件 | 功能 |
|------|------|
| LogManager | 多级别日志管理（DEBUG/INFO/WARNING/ERROR/CRITICAL） |
| Performance | 性能监控（推理时间、内存使用、QPS） |
| ErrorTracker | 错误追踪和异常记录 |
| StatsExport | 统计信息导出 |

## 三、数据流图

```
客户端请求
    ↓
FastAPI路由
    ↓
请求验证 (Pydantic)
    ↓
业务逻辑处理
    ├─→ 数据预处理
    ├─→ 模型推理
    └─→ 结果后处理
    ↓
响应序列化
    ↓
日志记录
    ↓
返回响应
```

## 四、API请求/响应示例

### 请求示例

```json
// POST /inference
{
  "opt_sequence": [[[[0.1, 0.2], [0.3, 0.4]], ...]],
  "sar_sequence": [[[[0.1, 0.2], [0.3, 0.4]], ...]],
  "dem_data": [[[0.1, 0.2], [0.3, 0.4]]],
  "doy": [0.1, 0.2, 0.3],
  "n_passes": 3,
  "use_tta": false
}
```

### 响应示例

```json
{
  "success": true,
  "message": "推理成功",
  "prediction": [[0, 1, 2], [1, 2, 0]],
  "probabilities": [[[0.9, 0.1], [0.2, 0.8]], ...],
  "uncertainty": {
    "vacuity": [[0.1, 0.2], [0.3, 0.4]],
    "dissonance": [[0.05, 0.1], [0.15, 0.2]]
  },
  "inference_time": 0.123,
  "model_version": "v5_edl"
}
```
