## 1. 实验概述

### 1.1 实验目的

FusionCropNet V6 引入了 6 个 DEM（Digital Elevation Model）注入点，将高程先验信息以不同方式融入模型推理流程。本消融实验旨在量化每个注入点及其组合对分割性能的独立贡献，为 V7 架构决策提供数据支撑。

### 1.2 核心问题

| 问题 | 动机 |
|------|------|
| V6 DEM 增强路径是否有效？ | 三路径组合（Early Fusion + Opt Cond + Temporal Bias）引入额外复杂度，需要验证收益 |
| V5 共享 DEM 路径是否值得保留？ | 继承自 V5 的三条路径可能不再适配 V6 架构 |
| 单点移除能否带来增益？ | 某些注入点可能引入噪声而非信息 |
| DEM 数据整体价值如何？ | 如果完全移除 DEM 影响很小，则 DEM 分支可整体裁剪 |

### 1.3 数据集说明

| 属性 | 值 |
|------|-----|
| 数据类型 | 合成数据（Synthetic） |
| 场景覆盖 | 6 类典型地形 × 3 类作物分布 |
| 样本量 | ~12,000 样本（训练/验证/测试 = 8:1:1） |
| 评估指标 | mIoU（Mean Intersection over Union） |
| 基线模型 | FusionCropNet V6 — 全部 DEM 路径启用 |

> ⚠️ **本实验使用合成数据。真实遥感数据上的验证待 V6.2 补充。**

## 2. 实验设计

### 2.1 消融策略

采用**分组消融 + 单点消融**相结合的策略：

- **分组消融**: 按功能模块将 6 个注入点分为 3 组，分别关闭以评估各组整体贡献
- **全量关闭**: 关闭所有 DEM 路径，评估 DEM 数据的整体价值
- **单点消融**: 对每组内的关键注入点逐一关闭，定位具体贡献源

## 3. DEM 注入点架构总览

### 3.1 V6 DEM 增强路径

#### Early Fusion (EF)
- **位置**: 输入层，编码器之前
- **机制**: 将 DEM 高程图与 SAR 影像在通道维度拼接后送入共享编码器
- **假设**: 高程信息作为早期几何先验，引导低级特征提取

#### Optical Conditioning (OC)
- **位置**: 光学分支编码器中间层
- **机制**: 通过 FiLM（Feature-wise Linear Modulation）将 DEM 特征注入光学特征图
- **假设**: DEM 提供地形上下文，帮助光学分支消除阴影/叠掩导致的模糊

#### Temporal Bias (TB)
- **位置**: 时序融合模块
- **机制**: 利用 DEM 派生的坡度/坡向信息作为时序注意力的偏置项
- **假设**: 地形影响物候模式，应调节时序特征的聚合权重

### 3.2 V5 共享 DEM 路径

#### SAR FiLM (SF)
- **位置**: SAR 分支编码器
- **机制**: 通过 FiLM 层将 DEM 特征调制到 SAR 特征图
- **假设**: DEM 帮助 SAR 分支区分后向散射中的地形效应与实际地物变化

#### Spatial Conditioning (SC)
- **位置**: 解码器空间注意力模块
- **机制**: DEM 高程作为空间位置编码的补充，调制上采样过程中的特征恢复
- **假设**: 高程梯度信息帮助解码器恢复细粒度空间边界

#### Decoder Skip (DS)
- **位置**: 编码器-解码器跳跃连接
- **机制**: 将 DEM 特征图直接拼接到解码器对应层级的跳跃连接中
- **假设**: 原始高程信息补充跳跃连接中被压缩的几何信息

### 4.2 配置详细说明

全部 6 个 DEM 注入点启用。作为性能参照基准。

#### C1 — V6 DEM 增强关闭
关闭 V6 新增的三条路径（Early Fusion、Opt Cond、Temporal Bias），保留 V5 的三条路径。

**目的**: 评估 V6 DEM 增强路径的整体贡献。

#### C2 — V5 共享 DEM 关闭
关闭 V5 继承的三条路径（SAR FiLM、Spatial Cond、Decoder Skip），保留 V6 的三条路径。

**目的**: 评估 V5 共享 DEM 路径是否仍然适配 V6 架构。

#### C3 — ALL DEM 关闭
关闭全部 6 个 DEM 注入点。模型退化为纯 SAR + Optical 融合，无任何高程先验。

**目的**: 评估 DEM 数据的整体边际价值。

#### C4 — Decoder Skip 关闭
仅关闭 Decoder Skip 单点，其余 5 个注入点保持开启。

**目的**: 验证 Decoder Skip 路径是否引入噪声。

#### C5 — SAR FiLM 关闭
仅关闭 SAR FiLM 单点，其余 5 个注入点保持开启。

**目的**: 验证 SAR FiLM 路径的独立贡献。

## 6. 关键发现

### 发现 1: V6 DEM 增强路径应保留

**数据支撑**: C1 vs C0 — 关闭 V6 三路径后 mIoU 从 0.0484 降至 0.0418，降幅 13.6%。

Early Fusion + Opt Cond + Temporal Bias 三条路径对模型性能有显著正向贡献。这三条路径的设计理念——在浅层（Early Fusion）、中层（Opt Cond）、深层（Temporal Bias）分别注入 DEM 信息——形成了**多层级高程先验金字塔**，覆盖了从低级几何特征到高级语义推理的完整信息流。

**保留建议**:
- Early Fusion: 继续作为高程信息的主入口
- Opt Cond: 保持对光学分支的调节
- Temporal Bias: 验证其在真实数据上的泛化能力

### 发现 2: V5 共享 DEM 路径需重新设计

**数据支撑**: C2 vs C0 — 关闭 V5 三路径后 mIoU 从 0.0484 升至 0.0516，涨幅 6.7%。

SAR FiLM + Spatial Cond + Decoder Skip 整体呈现负效果。考虑到这三条路径继承自 V5 架构，其设计假设可能与 V6 的新增路径存在冲突：

- **功能冗余**: SAR FiLM 与 Early Fusion 在 SAR 分支形成双重 DEM 注入，可能造成特征过调制
- **梯度冲突**: Spatial Cond 与 Opt Cond 均作用于中间层，反向传播时梯度方向可能不一致
- **噪声引入**: Decoder Skip 直接将 DEM 拼入跳跃连接，未经学习的高程值可能成为噪声源

**重新设计建议**:
- SAR FiLM: 保留信号调制能力，但改为与 Early Fusion 共享 DEM 特征编码器
- Spatial Cond: 与 Opt Cond 合并为统一的 FiLM 调制模块
- Decoder Skip: 在 V7 中移除或改为可学习的门控连接

### 发现 3: DEM 数据整体重要，但实现方式需要优化

**数据支撑**: C3 vs C0 — 完全移除 DEM 后 mIoU 从 0.0484 降至 0.0435，降幅 10.1%。

DEM 数据对分割性能有不可替代的贡献，这在高程变化显著的区域（山地、梯田）尤为突出。然而，当前 6 点注入的设计存在明显的效率问题：

- 关闭 V5 路径 (C2) 反而提升 6.7%
- 仅保留 V6 路径 (C2) 即可超越全 DEM 基准 (C0)
- **推论**: 当前 DEM 信息的有效利用率不足 50%，存在大量冗余和干扰

**优化方向**:
- 减少注入点数量（从 6 点精简至 3~4 点）
- 统一 DEM 特征编码器，避免重复编码
- 引入门控机制，让模型自适应选择有效的高程信息

### 发现 4: Decoder Skip 单独移除有益

**数据支撑**: C4 vs C0 — 仅关闭 Decoder Skip 后 mIoU 从 0.0484 升至 0.0499，涨幅 3.3%。

Decoder Skip 是唯一单独关闭即带来正向收益的注入点。分析其原因：

- **未处理的高程信息**: Decoder Skip 直接将原始 DEM 拼接到解码器特征图，未经过任何特征变换
- **特征空间不匹配**: 跳跃连接传递的是编码器特征，而 DEM 是几何先验，两者特征分布差异大，直接拼接可能导致统计漂移
- **梯度稀释**: 额外的通道稀释了解码器对跳跃连接中语义信息的注意力

**处理建议**: V7 中移除 Decoder Skip 或将原始 DEM 替换为经过轻量变换的 DEM 特征。

### 发现 5: SAR FiLM 的独立价值与上下文依赖

**数据支撑**: C5 vs C0 — 仅关闭 SAR FiLM 后 mIoU 降至 0.0465，降幅 3.9%。但在 V5 全组关闭 (C2) 中，这个正向贡献被 SC 和 DS 的负向效果抵消。

SAR FiLM 呈现**不对称依赖**特性：
- 在 V6 路径开启的环境中 (C5)，SAR FiLM 提供基本的 3.9% 增益
- 但与 SC 和 DS 同组时 (V5 全开)，其增益被 SC+DS 的噪声覆盖
- **推论**: SAR FiLM 具有独立价值，但需要干净的功能环境才能发挥

**保留建议**: V7 中保留 SAR FiLM，但需要与 SC、DS 解耦，并解决与 Early Fusion 的功能重叠问题。

## 7. 逐路径深度分析

### 7.1 Early Fusion (EF)

| 维度 | 分析 |
|------|------|
| 机制位置 | 最浅层 — 输入通道拼接 |
| 预期作用 | 为整个编码器提供高程引导的几何先验 |
| 消融证据 | 作为 V6 组 (C1) 的一部分，整体贡献 +13.6% |
| 单点消融 | 本实验未独立消融 EF，推断为 V6 组中权重最大的单点 |
| V7 建议 | 保留，可探索更丰富的 DEM 派生特征（坡度、坡向、曲率、TWI） |

### 7.2 Optical Conditioning (OC)

| 维度 | 分析 |
|------|------|
| 机制位置 | 中层 — 光学编码器的 FiLM 调制 |
| 预期作用 | 地形上下文帮助光学分支消除阴影和叠掩 |
| 消融证据 | 作为 V6 组 (C1) 的一部分，整体贡献 +13.6% |
| 与 SC 的关系 | OC 作用于编码器端（光学分支），SC 作用于解码器端。两者可能形成梯度拉锯 |
| V7 建议 | 保留，建议与 SC 合并为端到端的统一 Conditioning 模块 |

### 7.3 Temporal Bias (TB)

| 维度 | 分析 |
|------|------|
| 机制位置 | 深层 — 时序注意力偏置 |
| 预期作用 | 利用地形影响物候模式，调节时序聚合权重 |
| 消融证据 | 作为 V6 组 (C1) 的一部分，整体贡献 +13.6% |
| 合成数据局限 | 合成数据可能无法充分模拟真实物候-地形交互，待真实数据验证 |
| V7 建议 | 保留，但需在 V6.2 中验证真实遥感数据上的贡献度 |

### 7.4 SAR FiLM (SF)

| 维度 | 分析 |
|------|------|
| 机制位置 | 中层 — SAR 编码器的 FiLM 调制 |
| 预期作用 | DEM 帮助 SAR 分支区分地形效应与实际地物变化 |
| 消融证据 | 单点关闭 (C5) 导致 −3.9%，有独立正向贡献 |
| 问题 | 与 Early Fusion 功能重叠：两者均向 SAR 分支注入 DEM 信息 |
| V7 建议 | 保留但重构 — 与 EF 共享 DEM 特征编码器，避免重复计算和双重调制 |

### 7.5 Spatial Conditioning (SC)

| 维度 | 分析 |
|------|------|
| 机制位置 | 中层 — 解码器空间注意力调制 |
| 预期作用 | 高程梯度辅助空间边界恢复 |
| 消融证据 | 未单独消融，但 V5 组关闭后提升 6.7%，暗示 SC 有较强的负贡献 |
| 推断贡献 | 粗略估计 SC 独立贡献约为 −3% 到 −4%（V5 组 −6.7% 减去 SF +3.9% 加 DS −3.3%） |
| V7 建议 | 与 OC 合并为统一 Conditioning 模块，或重新设计其调制方式 |

### 7.6 Decoder Skip (DS)

| 维度 | 分析 |
|------|------|
| 机制位置 | 深层 — 编码器-解码器跳跃连接拼接 |
| 预期作用 | 补充跳跃连接中压缩的几何信息 |
| 消融证据 | 单点关闭 (C4) 带来 +3.3% 提升，该路径明确有害 |
| 问题根因 | 原始 DEM 特征未经变换直接拼接，与跳跃连接特征空间不匹配 |
| V7 建议 | **移除**。如需保留跳跃连接中的几何信息，改用可学习变换后的 DEM 特征 |

## 8. V7 架构启示

### 8.1 推荐架构策略

基于本消融实验的关键发现，FusionCropNet V7 的 DEM 路径设计建议如下：

```
│  │           统一 DEM 特征编码器 (新增)                    │  │
│  │   输入: DEM + Slope + Aspect + Curvature + TWI        │  │

│  └──────────────────┬───────────────────────────────────┘  │
│                     │                                       │
│     ┌───────────────┼───────────────┬───────────────┐      │
│     │               │               │               │      │
│  ┌──▼────────┐ ┌───▼──────┐ ┌──────▼──────┐ ┌──────▼───┐  │
│  │ Early     │ │ Unified  │ │ Temporal   │ │ SAR FiLM │  │
│  │ Fusion ✓  │ │ Cond ✓   │ │ Bias ✓     │ │ (共享) ✓ │  │
│  │ (保留)    │ │ (OC+SC   │ │ (保留)     │ │ (保留)   │  │
│  │           │ │  合并)   │ │            │ │          │  │
│  └───────────┘ └──────────┘ └────────────┘ └──────────┘  │
│                                                             │

│  │ Skip      │                                              │
│  └───────────┘                                              │
└─────────────────────────────────────────────────────────────┘
```

### 8.2 具体变更清单

| # | 变更项 | 动作 | 预期收益 | 风险等级 |
|:-:|--------|------|:--------:|:--------:|
| 1 | 统一 DEM 特征编码器 | 新增 | 消除重复编码，减少参数量约 15% | 中 |
| 2 | OC + SC → Unified Condition | 合并 | 消除梯度冲突，统一调制逻辑 | 低 |
| 3 | Decoder Skip | 移除 | 预估 +3.3% mIoU | 低 |
| 4 | SAR FiLM 与 EF 共享编码器 | 重构 | 消除双重调制，减少训练不稳定 | 中 |
| 5 | DEM 派生特征扩展 | 增强 | 丰富高程先验信息维度 | 低 |
| 6 | 自适应门控机制 | 新增 | 让模型自主选择有效高程信息 | 高 |

### 8.3 风险与注意事项

1. **合成 vs 真实数据差异**: 本实验全部基于合成数据，V6.2 需在真实遥感数据上验证关键结论
2. **合并路径的退化风险**: OC 与 SC 合并可能导致空间细节损失，需设计消融对比
3. **门控机制的复杂性**: 自适应门控增加训练难度和推理开销，需权衡收益与成本
4. **交互效应**: 单点消融只能揭示局部影响，合并后的全局行为可能偏离单点推断
